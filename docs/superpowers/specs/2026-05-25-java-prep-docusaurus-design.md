# Design Spec: java_prep Docusaurus Website

**Date:** 2026-05-25  
**Status:** Approved  
**Author:** Claude (brainstorming session)

---

## Overview

Build a public-facing Docusaurus 3.x documentation website in `notes_web/` that publishes the content from `../java_prep` — a comprehensive software developer interview preparation knowledge base with 200+ markdown files across 10+ topic areas.

## Goals

- Publish `java_prep` content as a searchable, navigable public website
- Host on GitHub Pages using Docusaurus's built-in deploy command
- Mirror the existing folder structure for the sidebar (zero reorganization)
- Copy markdown files into the Docusaurus repo (self-contained, no runtime dependency on `java_prep`)

## Non-Goals

- Restructuring or rewriting content
- Authentication / access control
- Algolia full-text search (can be added later)
- Custom theming beyond basic config

### Content Source

- Source: `/Users/neosoft/IdeaProjects/java_prep/`
- All `.md` files copied verbatim into `docs/`
- `README.md` files in each subfolder become the section index page
- Numeric folder prefixes (`01-java`, `07-dsa`) drive sidebar ordering automatically
- `_category_.json` files added per folder to control human-readable sidebar labels

### Docusaurus Config Key Decisions

| Setting | Value |
|---|---|
| Site title | "Software Dev Interview Prep" |
| Tagline | "A comprehensive guide for senior engineering interviews" |
| Base URL | `/notes_web/` (GitHub Pages project site) |
| Sidebar | `autogenerated` from `docs/` directory |
| Navbar | Logo + "Docs" link + GitHub icon link |
| Footer | Simple copyright line |
| Preset | `@docusaurus/preset-classic` |

### GitHub Pages Deployment

- `organizationName` and `projectName` set in `docusaurus.config.js`
- Deploy via: `npm run deploy` (uses `docusaurus deploy` internally)
- Publishes to `gh-pages` branch of the repo
- Requires the GitHub repo to have GitHub Pages enabled pointing at `gh-pages` branch

## Content Handling

### Markdown Compatibility

Docusaurus renders standard CommonMark Markdown. The existing `java_prep` files use:
- Headings, bullet lists, code blocks — all render natively
- Internal relative links (e.g., `./01-ioc-and-di.md`) — Docusaurus resolves these correctly
- Front matter (`---`) — none present in source files; Docusaurus handles files without it gracefully

### Sidebar Labels

Each folder gets a `_category_.json` with a human-readable label:

| Folder | Label |
|---|---|
| `00-prerequisite` | Prerequisites |

### Landing Page

`src/pages/index.js` — a simple hero page with:
- Title and tagline
- Brief description of what the site covers
- Grid of section links (one card per topic area)
- Link to the study guide / README

## Implementation Steps

1. Scaffold Docusaurus 3.x with `create-docusaurus` classic template in `notes_web/`
2. Copy all `java_prep` markdown files into `docs/` preserving folder structure
3. Add `_category_.json` to each top-level folder with human-readable label
4. Update `docusaurus.config.js` with site title, GitHub Pages settings, navbar
5. Update `sidebars.js` to use `autogenerated`
6. Build a simple custom landing page (`src/pages/index.js`)
7. Run `npm run build` to verify no broken links / build errors
8. Initialize git repo, push to GitHub, enable GitHub Pages
9. Run `npm run deploy` to publish

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Broken internal links between `.md` files | Docusaurus link checker flags these at build time |
| Large number of files slows build | Docusaurus handles 200+ files fine; no action needed |
| `README.md` files used as category index may have wrong titles | Add front matter `title:` to override if needed |
| GitHub repo name must match `projectName` in config | Document clearly in step-by-step plan |
