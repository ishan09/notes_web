# Java 12-17 Features (The Productivity Era)

> **Before you start**: Understand how `var` works from Java 10? → [Review Java 9-11](./java9-11.md)

## What changed in Java 12-17?

This range focuses on **Developer Joy**. Java 17 (LTS) consolidated many "Preview" features into the standard language, making Java code much denser, safer, and more readable.

## Core Concepts

### 1. Records (Java 14+)
A concise way to create "data carrier" classes (POJOs).

**Why This Matters**: It removes 90% of boilerplate. A Record automatically gets getters, `equals()`, `hashCode()`, and `toString()`.

```java
// Equivalent to ~50 lines of boilerplate
public record User(String name, int age) {}
```

**Try it yourself**: Create a Record and see if you can modify one of its fields (Hint: Records are immutable).

### 2. Sealed Classes (Java 15+)
Restricting which classes can extend or implement a class/interface.

**Analogy**: Like a security guard at a VIP club - only classes on the `permits` list are allowed to extend the base class.

### 3. Text Blocks (Java 13+)
Multi-line string literals that look natural and handle indentation automatically.

```java
String json = """
    {
        "name": "Java",
        "version": 17
    }
    """;
```
